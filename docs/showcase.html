route Pipeline

Step-by-Step Data Flow

From raw healthcare messages to analytics-ready tables in eight automated steps.

Load Source Data

Consumer loads HL7v2 messages, FHIR JSON, or FHIR XML bundles into any Snowflake table.

Auto-Detect Format

App inspects the first bytes of each record — MSH|, {, or < — to classify the format.

Parse to FHIR R4

HL7v2 segments are converted to FHIR resources. JSON/XML bundles are parsed directly. Batch inserts of 500.

Store Resources

Normalized FHIR R4 resources stored in app_state.fhir_resources with type, ID, and JSON payload.

Select Output Model

User chooses OMOP CDM v5.4 (research), Tuva Input Layer (analytics), or both via Streamlit UI.

Run Mapper Pipeline

23 OMOP or 13 Tuva mappers execute in sequence with real-time progress bar and per-step timing.

Create Output Tables

Output tables materialized in user-specified schema. Vocabulary crosswalks resolve terminology codes.

Analytics Ready

Query OMOP tables with OHDSI tools or feed Tuva tables into dbt for HCC, CCSR, readmissions, PMPM.
